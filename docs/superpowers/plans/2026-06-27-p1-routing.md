# P1 ‚Äî Routing + Role + MerchantContext (expanded)

> Subagent-driven, run-and-verify. Builds on P0 (schema/RLS done). Goal: add react-router + a session/role layer + merchant-by-slug context WITHOUT breaking the existing single-tenant experience (which stays at `/`). New routes `/s/:slug`, `/merchant/*`, `/admin/*` are additive placeholders that P2‚ÄìP6 fill.

**Architecture:** Lift auth (account + profile + role + lang) into a `SessionProvider` mounted above a `<Routes>` tree, so route-level components outside the legacy `App` can read role for guarding. Legacy `App` keeps rendering the current owner/customer experience at `/`, but its hardcoded `USER_EMAIL` gate is replaced by `app_role` from the session. `MerchantProvider` resolves `:slug ‚Üí merchant` for storefront routes.

**Verify:** `npm run dev` loads; existing login/owner/customer flows still work at `/`; `/s/<anything>` shows a merchant shell or "shop not found"; `/merchant` and `/admin` redirect when unauthorized. `npm test` stays green; store unit tests added.

## Global Constraints
- React 19.2.x / Vite 8.x; ESM. All Supabase access via `src/store.js`.
- Do not break the existing `/` experience (owner + customer). Transitional: keep `USER_EMAIL` as a fallback that maps to `superadmin` until P3 seeds real roles ‚Äî clearly marked for removal.
- `t(en, zh)` preserved.
- `RESERVED_SLUGS` (src/slug.js) already lists router top-level segments ‚Äî `/s/:slug` resolution must treat reserved slugs as not-a-merchant.

### Task 1.3: SessionProvider ‚Äî lift auth/profile/role/lang

  - `profile`: the row from `fetchProfileByUserId` (or null).
  - `role`: `profile?.app_role ?? (account?.email === USER_EMAIL ? 'superadmin' : 'customer')` ‚Äî transitional email fallback, marked for removal in P3.
  - `t`: `(en, zh) => lang === 'zh' ? zh : en`.

- [ ] **Step 1: Create `src/SessionContext.jsx`**

```jsx
import { createContext, useContext, useEffect, useState, useCallback } from 'react'
import { onAuthChange, fetchProfileByUserId } from './store'

// TODO(P3): remove this transitional fallback once superadmin role is seeded in DB.
const USER_EMAIL = 'bitetimeandco@gmail.com'

const SessionContext = createContext(null)

export function SessionProvider({ children }) {
  const [account, setAccount] = useState(undefined)
  const [profile, setProfile] = useState(null)
  const [lang, setLang] = useState('en')

  const loadProfile = useCallback(async (user) => {
    if (!user) { setProfile(null); return }
    try { setProfile(await fetchProfileByUserId(user.id)) }
    catch { setProfile(null) }
  }, [])

  useEffect(() => {
    const sub = onAuthChange((user) => {
      setAccount(user ?? null)
      loadProfile(user)
    })
    return () => { sub?.data?.subscription?.unsubscribe?.() }
  }, [loadProfile])

  const role = profile?.app_role
    ?? (account?.email === USER_EMAIL ? 'superadmin' : 'customer')

  const t = (en, zh) => (lang === 'zh' ? zh : en)
  const refreshProfile = () => loadProfile(account)

  const value = { account, profile, role, loading: account === undefined, lang, setLang, t, refreshProfile }
  return <SessionContext.Provider value={value}>{children}</SessionContext.Provider>
}

export function useSession() {
  const ctx = useContext(SessionContext)
  if (!ctx) throw new Error('useSession must be used within SessionProvider')
  return ctx
}
```

- [ ] **Step 1: `src/MerchantContext.jsx`**

```jsx
import { createContext, useContext, useEffect, useState } from 'react'
import { useParams } from 'react-router-dom'
import { fetchMerchantBySlug } from './store'

const MerchantContext = createContext({ merchant: null, loading: true, notFound: false })

export function MerchantProvider({ children }) {
  const { slug } = useParams()
  const [state, setState] = useState({ merchant: null, loading: true, notFound: false })
  useEffect(() => {
    let on = true
    setState({ merchant: null, loading: true, notFound: false })
    fetchMerchantBySlug(slug).then((m) => {
      if (on) setState({ merchant: m, loading: false, notFound: !m })
    })
    return () => { on = false }
  }, [slug])
  return <MerchantContext.Provider value={state}>{children}</MerchantContext.Provider>
}

export const useMerchant = () => useContext(MerchantContext)
```
